The Tester role is deterministic in code: it extracts runnable shell
commands from each task's Evidence Required section, runs them in the
project directory, records the actual output in Evidence Log, and writes
TEST_REPORT.md.

This prompt file is retained only for role documentation and backwards
compatibility with prompt-loading tooling. Do not use an LLM to verify
Coder claims. Evidence must come from commands the framework actually
ran.
